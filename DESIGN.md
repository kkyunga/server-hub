🛠️ 5번째 탭: 운영 및 배포 (Management & Deploy)
화면 구성은 크게 상단(배포 설정), 중앙(실시간 상태), 하단(히스토리 및 롤백) 3단 레이아웃으로 제안합니다.

1. [상단] 배포 방식 및 소스 설정 (Deploy Setup)
사용자가 현재 어떤 환경이든 대응할 수 있도록 선택지를 줍니다.

배포 모드 선택 (Tab Switch): [ GitHub ] [ 사내 Git ] [ 수동 업로드 ]

Git 모드: Repo URL, 브랜치명, Access Token 입력 필드 노출.

수동 모드: 파일을 직접 올릴 수 있는 드래그 앤 드롭 존 노출.

빌드 스크립트 (Editor): 배포 시 실행할 명령어 입력 (예: npm install && npm run build).

액션 버튼: 우측 하단에 위치한 강력한 [ 실시간 배포 시작 ] (초록색 버튼).

2. [중앙] 실시간 파이프라인 & 콘솔 (Live Pipeline & Console)
배포 버튼을 누르는 순간 활성화되며, 실시간으로 진행 상황을 보여줍니다.

파이프라인 비주얼라이저:

소스 확보 → 의존성 설치 → 빌드/컴파일 → 서비스 재시작 → 완료

진행 중인 단계는 빙글빙글 도는 스피너, 완료는 체크 표시, 실패는 빨간 X로 표시.

실시간 로그 콘솔: 터미널 탭의 UI를 작게 가져와 배포 로그를 한 줄씩 출력 (Socket.io 연동 권장).

3. [하단] 배포 히스토리 및 롤백 (History & Rollback)
과거의 기록을 관리하고, 문제가 생겼을 때 즉시 되돌리는 '보험' 영역입니다.

롤백(Rollback) 버튼: 주황색 또는 테두리 버튼으로 배치하여 클릭 시 "이 시점으로 서버 소스를 되돌리시겠습니까?" 팝업을 띄웁니다.

백업(Backup) 연동: 롤백 버튼을 누르면 현재 운영 중인 소스를 자동으로 백업한 뒤, 선택한 과거 시점의 데이터를 복구합니다.
